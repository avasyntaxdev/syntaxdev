شرح Laravel Middleware خطوة بخطوة (مع مثال عملي)

الـ Middleware في Laravel يعمل كطبقة فلترة تمر من خلالها الطلبات (Requests) قبل وصولها إلى الـ Controller. يستخدم عادةً للتحقق من تسجيل الدخول، الصلاحيات، أو أي شرط قبل عرض الصفحة.

ما هو Middleware في Laravel ؟

يمكن اعتباره حارساً يقف بين المستخدم والتطبيق. عند إرسال طلب إلى Route معين، يتم تنفيذ الـ Middleware أولاً، فإذا تحقق الشرط يسمح بمرور الطلب، وإذا لم يتحقق يتم منعه أو إعادة توجيهه.

إنشاء Middleware جديد


php artisan make:middleware CheckUser

سيتم إنشاء الملف داخل: 


app/Http/Middleware/CheckUser.php

كتابة منطق التحقق داخل Middleware


public function handle($request, Closure $next)
{
    if(!auth()->check()){
        return redirect('/login');
    }

    return $next($request);
}

إذا لم يكن المستخدم مسجلاً سيتم تحويله إلى صفحة تسجيل الدخول.

تسجيل Middleware داخل Kernel

افتح الملف: 


app/Http/Kernel.php

وأضف داخل $routeMiddleware: 


'checkuser' => \App\Http\Middleware\CheckUser::class,

استخدام Middleware داخل Route


Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware('checkuser');

الآن لن يتمكن أي مستخدم غير مسجل من الدخول إلى dashboard.

استخدام Middleware لمجموعة Routes


Route::middleware(['checkuser'])->group(function () {
    Route::get('/dashboard', function () {
        return view('dashboard');
    });
});

استخدام Middleware الجاهز في Laravel

Laravel يحتوي على Middleware جاهز مثل:

  • auth
  • verified
  • throttle
