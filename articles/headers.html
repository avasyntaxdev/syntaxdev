حل مشكلة headers already sent في PHP (شرح السبب + الحل النهائي)

تعتبر مشكلة Warning: Cannot modify header information - headers already sent من أكثر الأخطاء انتشاراً بين مطوري PHP، خصوصاً عند استخدام session_start() أو تنفيذ تحويل باستخدام header().

كيف تظهر رسالة الخطأ؟


Warning: Cannot modify header information - headers already sent

تظهر غالباً عند محاولة تنفيذ redirect أو بدء جلسة بعد إرسال أي إخراج للمتصفح.

ما معنى الخطأ تقنياً؟

بروتوكول HTTP يفرض إرسال معلومات الهيدر (Headers) أولاً قبل أي محتوى HTML. إذا تم إرسال أي نص — حتى مسافة فارغة أو سطر جديد — يعتبر أن الإخراج بدأ، وبالتالي لا يمكن تعديل الهيدر بعد ذلك.

أسباب حدوث المشكلة

  • وجود مسافة قبل بداية كود PHP <?php
  • استخدام echo قبل header()
  • وجود HTML قبل session_start()
  • إغلاق ملف PHP بعلامة ?> ثم إضافة سطر جديد بعدها
  • حفظ الملف بترميز يحتوي على BOM

مثال يسبب الخطأ


echo "Welcome";  

header("Location: home.php");

استخدام session_start بشكل صحيح

حل مؤقت باستخدام Output Buffering

تقوم الدالة ob_start() بتأجيل الإخراج مؤقتاً، لكن يفضل إصلاح السبب الأساسي بدلاً من الاعتماد عليها.

أفضل ممارسة لتجنب المشكلة نهائياً

  • ضع session_start() في أول سطر في الملف
  • لا تستخدم ?> في نهاية ملفات PHP البحتة
  • تأكد من عدم وجود مسافات قبل <?php
  • لا تستخدم echo قبل عمليات redirect

أسئلة شائعة حول الخطأ

لماذا يظهر الخطأ رغم أن الكود صحيح؟

غالباً يكون السبب وجود مسافة مخفية أو مشكلة في ترميز الملف (BOM).

هل يمكن تجاهل الخطأ؟

لا، لأنه يمنع تنفيذ التحويل أو بدء الجلسة بشكل صحيح.

هل ob_start() حل نهائي؟

هو حل مؤقت، لكن الأفضل دائماً إصلاح سبب الإخراج المبكر.
