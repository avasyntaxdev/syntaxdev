شرح Laravel Routing خطوة بخطوة (مع أمثلة عملية)

الـ Routing في Laravel هو المسؤول عن ربط الرابط (URL) بالوظيفة المناسبة داخل التطبيق سواء كانت Closure أو Controller. بدون Routing لن يستطيع Laravel معرفة أي كود يجب تنفيذه عند زيارة رابط معين.

أين يوجد ملف Routes في Laravel؟

ستجد ملفات التوجيه داخل مجلد: 


routes/web.php

هذا الملف مسؤول عن Routes الخاصة بالويب.

مثال بسيط على Route


Route::get('/home', function () {
    return view('home');
});

عند زيارة الرابط: 


/home

سيتم عرض صفحة home.

أنواع الـ Routes في Laravel

  • Route::get()
  • Route::post()
  • Route::put()
  • Route::delete()
  • Route::any()

كل نوع يستخدم حسب طريقة الطلب (HTTP Method).

تمرير متغير داخل الرابط (Route Parameters)


Route::get('/user/{id}', function ($id) {
    return "User ID: " . $id;
});

الآن عند زيارة: 


/user/5

سيظهر: 


User ID: 5

جعل المتغير اختياري


Route::get('/user/{name?}', function ($name = "Guest") {
    return $name;
});

استخدام Controller بدلاً من Closure


Route::get('/dashboard', [DashboardController::class, 'index']);

وهذا أفضل في المشاريع الكبيرة لتنظيم الكود.

تجميع Routes باستخدام Group


Route::prefix('admin')->group(function () {
    Route::get('/dashboard', function () {
        return view('admin.dashboard');
    });
});

الآن الرابط سيكون: 


/admin/dashboard

أفضل ممارسات تنظيم Routing

  • استخدم Controllers بدلاً من Closures في المشاريع الكبيرة
  • استخدم Route Groups لتنظيم المسارات
  • ضع Middleware لحماية الصفحات
